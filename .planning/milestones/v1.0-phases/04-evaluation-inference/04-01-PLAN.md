  <action>
    **In src/evaluation/metrics.py**, add three functions (keep existing spans_to_bio and compute_entity_metrics unchanged):

    1. `span_iou(pred: tuple[int,int], gold: tuple[int,int]) -> float` — intersection over union for character-offset spans. Handle zero-length union by returning 1.0.

    2. `classify_span_type(span_text: str, full_text: str = "") -> str` — classify a span's reference type by matching the span text against individual regex patterns. Return one of: "PARAGRAPH" (§/§§), "ARTIKEL" (Art./Artikel), "ABSATZ" (Abs./Absatz), "NUMMER" (Nr./Nummer), "LITERAL" (lit.), "SATZ" (Satz/S.), "TEILZIFFER" (Tz./Teilziffer), "ANHANG" (Anhang), "VERORDNUNG" (Verordnung/EU-Verordnung), or "REF" (fallback). Use `regex` PyPI package (already imported in regex_baseline.py; import it here too). Each pattern is simple: just check if span_text starts with or contains the keyword.

    3. `decode_bio_to_char_spans(token_labels: list[int], offset_mapping: list[tuple[int,int]]) -> list[tuple[int,int]]` — reconstruct character spans from integer BIO labels + offset_mapping. Skip special tokens where start==0 and end==0. Handle B-REF starting new span, I-REF continuing, O/other closing. This generalizes the logic from `validate_bio_roundtrip` in bio_converter.py but returns spans instead of comparing them.

    4. `compute_partial_match_metrics(gold_spans: list[tuple[int,int]], pred_spans: list[tuple[int,int]], iou_threshold: float = 0.5) -> dict` — compute P/R/F1 at span level using IoU matching. A pred span is a TP if it matches any gold span with IoU > threshold (greedy 1:1 matching). Returns dict with precision, recall, f1 keys.

    **In src/evaluation/regex_baseline.py**, add to `RegexBaseline`:

    5. `TYPED_PATTERNS` class attribute: dict mapping type name (str) to compiled regex pattern. Split the monolithic `GERMAN_LEGAL_REF_PATTERN` into individual named patterns. Keep the existing monolithic pattern and `extract()` method unchanged for backward compatibility.

    6. `extract_typed(self, text: str) -> list[tuple[int, int, str]]` — iterate TYPED_PATTERNS, collect all matches with type name, sort by start position, deduplicate overlapping spans (keep longest).

    **In tests/test_evaluator.py**, write tests covering the behavior list above. Test classes: TestIoUScoring, TestSpanTypeClassification, TestBIODecode, TestPartialMatchMetrics, TestTypedExtraction. Use descriptive test names. Mock nothing — these are pure functions.

  <action>
    **Extend src/evaluation/evaluator.py** — add methods to the existing Evaluator class:

    1. `load_gold_set(self, path: str | Path | None = None) -> list[dict]` — load gold_test_set.json from `data/gold_test/gold_test_set.json` (default) or given path. Convert JSON array spans to tuples. Raise FileNotFoundError with clear message if missing.

    2. `evaluate_model(self, model, tokenizer, samples: list[dict], device: torch.device) -> dict` — for each gold sample:
       - Tokenize with `BertTokenizerFast` (passed as arg), return_offsets_mapping=True, truncation=True, padding="max_length", max_length from config
       - Run model.forward(input_ids, attention_mask) with torch.no_grad() and model.eval()
       - Handle both return types: non-CRF (output.logits.argmax(dim=-1)) and CRF (list[list[int]])
       - Build gold/pred BIO string lists filtering positions where gold label != -100
       - For per-type: use classify_span_type on gold span text to create typed BIO labels (B-PARAGRAPH, I-PARAGRAPH, etc.) then use seqeval classification_report with output_dict=True
       - For IoU: use decode_bio_to_char_spans on pred labels + offset_mapping to get pred char spans, then compute_partial_match_metrics against gold char spans
       - Return dict: precision, recall, f1, per_type (from classification_report), partial_match (IoU P/R/F1), report (seqeval text report)

    3. `evaluate_comparison(self, model, tokenizer, samples, device) -> dict` — run evaluate_model + evaluate_baseline on same samples, compute deltas, return {ml: metrics, baseline: metrics, delta: {precision, recall, f1}}

    4. `dump_errors(self, samples, pred_spans_per_sample, output_path)` — write JSON list of error records with fields: sample_idx, text, gold_spans, pred_spans, false_positives, false_negatives, domain

    5. `format_comparison_report(self, comparison: dict) -> str` — format the side-by-side comparison table as per research doc pattern: Metric / ML Model / Regex Baseline / Delta columns, plus verdict line about recall

    **Important implementation notes:**
    - Import torch, BertTokenizerFast at module level (both already in requirements)
    - Use `from src.data.bio_converter import LABEL_O, LABEL_B_REF, LABEL_I_REF, LABEL_IGNORE` for label constants
    - Use `from src.evaluation.metrics import decode_bio_to_char_spans, classify_span_type, compute_partial_match_metrics, span_iou` for new utilities
    - For per-type BIO: when building seqeval input, replace generic "B-REF"/"I-REF" with typed versions like "B-PARAGRAPH"/"I-PARAGRAPH" based on classify_span_type of the corresponding gold span
    - The gold test set has bio_labels with input_ids/attention_mask/labels — use these directly for gold labels. Tokenize the text separately for offset_mapping.
