**Create src/model/ner_model.py** (GREEN phase):
- Class `RegulatoryNERModel(nn.Module)` with `NUM_LABELS = 3` and label constants `O, B_REF, I_REF = 0, 1, 2`.
- `__init__(self, config)`: If `config.model.use_crf` is True, create `self.bert = BertModel.from_pretrained(config.model.name)`, `self.classifier = nn.Linear(hidden_size, NUM_LABELS)`, `self.crf = CRF(NUM_LABELS, batch_first=True)`, set `self._use_crf = True`. Else, create `self.bert_tc = BertForTokenClassification.from_pretrained(config.model.name, num_labels=NUM_LABELS)`, `self._use_crf = False`.
- If `config.model.freeze_backbone` is True: freeze all BERT encoder params via `requires_grad = False`. Do this BEFORE LoRA so LoRA can override on adapter params.
- If `config.model.use_lora` is True: apply PEFT LoRA with `LoraConfig(task_type=TaskType.TOKEN_CLS, r=config.model.lora_rank, lora_alpha=config.model.lora_rank * 2, target_modules=["query", "value"], lora_dropout=0.1, bias="none")`. Apply via `get_peft_model()` on the BERT submodule. Handle both CRF and non-CRF paths.
- `forward(self, input_ids, attention_mask, labels=None)`:
  - Non-CRF: delegate to `self.bert_tc(input_ids, attention_mask, labels=labels)` — returns TokenClassifierOutput with .loss and .logits.
  - CRF: get `outputs = self.bert(input_ids, attention_mask)`, compute `emissions = self.classifier(outputs.last_hidden_state)`. If labels provided: build `mask = (labels != -100).bool()`, clone labels, replace -100 with 0, compute `loss = -self.crf(emissions, clean_labels, mask=mask, reduction="mean")`, return `(loss, emissions)`. If labels is None: return `self.crf.decode(emissions, mask=attention_mask.bool())`.
- `property use_crf -> bool`: expose toggle state.
- `get_bert_parameters()` and `get_head_parameters()` methods for differential LR (Plan 02 will use these).

**Important:** For LoRA, if `target_modules=["query", "value"]` does not match gbert-large module names (open question from research), add a fallback: inspect `[n for n, _ in model.named_modules()]` and look for modules containing "query"/"value" or "q_proj"/"v_proj". Log a warning if no match and skip LoRA gracefully.
