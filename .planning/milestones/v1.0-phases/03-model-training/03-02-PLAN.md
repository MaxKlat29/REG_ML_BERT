**Create src/model/trainer.py** (GREEN phase):

**`resolve_mixed_precision(config, device) -> str`:**
- If device is CUDA: return config.training.mixed_precision (default "bf16" or whatever user set).
- If device is MPS: check torch version >= 2.6 for bf16 support, >= 2.8 for fp16; fall back to "no".
- If device is CPU: return "no".

**`build_optimizer(model, config) -> AdamW`:**
- Two param groups using `model.get_bert_parameters()` (lr=learning_rate_backbone) and `model.get_head_parameters()` (lr=learning_rate_head).
- Filter to only `requires_grad=True` params in each group.
- `weight_decay=0.01`.

**`build_scheduler(optimizer, config, steps_per_epoch) -> LambdaLR`:**
- `num_training_steps = config.training.num_epochs * steps_per_epoch`
- Use `get_linear_schedule_with_warmup(optimizer, num_warmup_steps=config.training.warmup_steps, num_training_steps=num_training_steps)`.

**`save_checkpoint(model, optimizer, scheduler, epoch, config, accelerator, run_id="") -> Path`:**
- Unwrap model via `accelerator.unwrap_model(model)`.
- Save to `checkpoints/{run_id}/epoch_{epoch}.pt` with keys: epoch, model_state_dict, optimizer_state_dict, scheduler_state_dict.
- Return the path.

**`load_checkpoint(path, model, optimizer=None, scheduler=None) -> int`:**
- Load state dict, restore model weights, optionally restore optimizer/scheduler.
- Return the epoch number.

**`class Trainer`:**
- `__init__(self, config, model, tokenizer, accelerator)`: store refs.
- `train(self) -> Path`: Main training loop:
  1. Build optimizer via `build_optimizer`.
  2. Estimate `steps_per_epoch = config.data.samples_per_batch` (approximate for IterableDataset; this is the number of batches per epoch from the LLM generator).
  3. Build scheduler via `build_scheduler`.
  4. `model, optimizer, scheduler = accelerator.prepare(model, optimizer, scheduler)`.
  5. For each epoch:
     a. Create `LLMGeneratedDataset(config, tokenizer, epoch=epoch)`.
     b. Create DataLoader with `batch_size=config.training.batch_size`.
     c. Prepare dataloader: `dataloader = accelerator.prepare(dataloader)`.
     d. For each batch: `accelerator.autocast()` context, forward pass, `accelerator.backward(loss)`, `accelerator.clip_grad_norm_(model.parameters(), config.training.max_grad_norm)`, optimizer.step(), scheduler.step(), optimizer.zero_grad().
     e. Log epoch loss.
     f. Save checkpoint via `save_checkpoint`.
  6. Return path to final checkpoint.

**Important pitfall handling:**
- Build optimizer BEFORE `accelerator.prepare()` — pass all 3 to prepare together.
- Recreate dataset + dataloader each epoch to get fresh seeds (Pitfall 5 from research).
- Use `BertTokenizerFast` not `AutoTokenizer` (Pitfall 6 from research).

**Add ensemble methods to src/model/trainer.py:**

**`majority_vote(predictions: list[list[int]]) -> list[int]`:**
- Takes list of N model predictions (each a list of int labels for one sequence).
- Returns per-position majority label using `collections.Counter.most_common(1)`.
- Handle ties by picking the first most-common (deterministic).

**`train_ensemble(config, tokenizer, accelerator) -> list[Path]`:**
- Outer loop for `range(config.ensemble.n_estimators)`.
- Model 0: `cache_path=None`, after training writes cache to `config.data.cache_dir / "ensemble_base.jsonl"`.
- Models 1..N-1: `cache_path = config.data.cache_dir / "ensemble_base.jsonl"` (reads from first model's cache).
- Each model gets seed offset: `config.project.seed + i * 1000`.
- Instantiate fresh `RegulatoryNERModel(config)` per estimator.
- Create `Trainer(config, model, tokenizer, accelerator)` and call `trainer.train()`.
- Collect and return checkpoint paths.
- If `config.ensemble.get("use_gradient_boost", False)` is True: log a warning "gradient-boost ensemble not yet implemented, falling back to uniform resampling" — this is the ENSM-04 stub.
